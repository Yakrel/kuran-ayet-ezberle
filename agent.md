# Development Notes (Agent)

- Android and iOS are the primary targets.
- Web is only for rough visual checks; do not prioritize web-specific behavior unless explicitly requested.
- Build policy: All official development and production builds MUST be produced by GitHub Actions (CI). Do not use Expo's build service or local builds as a replacement for CI-produced release artifacts.
- Local development: Use the local dev server and dev client only for interactive development and manual testing. Release/distribution artifacts must come from CI.
- If a native dependency or native config causes an error, a native rebuild is required.
- Do not add runtime fallbacks or workaround layers just to avoid a needed native rebuild — prefer correct native fixes.
- Use `.env` values when triggering remote builds or build-related commands.
- Normal JS/UI changes only need the dev server + hot reload. Native dependency/config changes require a new development build.

Policy:
- Commit messages and repository metadata MUST NOT contain AI-identifying trailers or markers (e.g., "Co-authored-by: Copilot", "Copilot", "AI", "Generated by <AI>"). If such trailers exist in history, coordinate with the repository administrator to remove them from the public history.
